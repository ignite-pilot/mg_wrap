# 자동 테스트 및 보안 체크 가이드

코드 수정 후 자동으로 테스트와 보안 체크를 수행하는 방법입니다.

## 자동 실행 방법

### 1. 수동 실행

코드 수정 후 다음 명령어를 실행하세요:

```bash
python run_tests_and_security.py
```

또는:

```bash
./run_tests_and_security.py
```

### 2. Git Pre-commit Hook 설정 (권장)

커밋 전에 자동으로 실행되도록 설정:

```bash
# .git/hooks/pre-commit 파일 생성 또는 수정
cp pre-commit-hook.sh .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```

이제 `git commit`을 실행하면 자동으로 테스트와 보안 체크가 실행됩니다.

### 3. GitHub Actions (자동)

GitHub에 푸시하면 자동으로 실행됩니다. `.github/workflows/test_and_security.yml` 파일이 설정되어 있습니다.

## 체크 항목

### 1. 보안 체크
- ✅ 하드코딩된 비밀번호 검사
- ✅ CORS 설정 확인 (모든 출처 허용 방지)
- ✅ SECRET_KEY 기본값 확인
- ✅ 에러 메시지에서 민감한 정보 노출 확인

### 2. 유닛 테스트
- ✅ 모든 테스트 케이스 실행
- ✅ 테스트 결과 리포트

### 3. 의존성 보안 체크 (선택사항)
- ✅ safety를 사용한 패키지 취약점 검사

## 문제 해결

### 테스트 실패 시
1. 테스트 결과를 확인하여 실패한 테스트 케이스 확인
2. 코드를 수정하여 테스트 통과하도록 수정
3. 다시 실행

### 보안 체크 실패 시
1. 발견된 보안 문제 확인
2. 해당 파일의 문제 부분 수정
3. 다시 실행

### 의존성 취약점 발견 시
```bash
# safety로 상세 정보 확인
safety check

# 취약한 패키지 업데이트
pip install --upgrade <package-name>
```

## CI/CD 통합

GitHub Actions를 사용하면:
- 모든 푸시와 Pull Request에서 자동 실행
- 테스트 실패 시 PR 병합 차단
- 결과를 GitHub에 표시

## 추가 설정

### safety 설치 (의존성 보안 체크용)
```bash
pip install safety
```

### pytest 옵션 커스터마이징
`pytest.ini` 파일에서 pytest 옵션을 수정할 수 있습니다.

## 주의사항

- 프로덕션 배포 전에는 반드시 모든 체크를 통과해야 합니다.
- 보안 체크는 기본적인 항목만 확인합니다. 추가 보안 검토가 필요할 수 있습니다.
- 테스트 커버리지를 높이기 위해 새로운 기능에 대한 테스트를 작성하세요.
